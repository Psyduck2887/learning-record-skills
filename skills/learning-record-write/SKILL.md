# Learning Record Write

只记录已经解决的问题。保持记录轻量、具体、可复用；除非用户明确要求，不写长篇复盘。

## 路径解析

1. 优先使用 `LEARNING_RECORD_REPO`。
2. 如果没有设置 `LEARNING_RECORD_REPO`，询问用户本地 `learning_record` 仓库路径。
3. 不要假设默认本机路径、GitHub 账号或仓库位置。

## 执行流程

1. 如果用户没有指定主题，询问要写入哪个主题。
2. 读取 `learning_record` 根目录 `README.md`。
3. 根据根目录 `README.md` 的主题列表和别名，找到主题目录。
4. 读取主题目录下的 `README.md`。
5. 严格按照主题 `README.md` 的写入规则，找到目标笔记文件和格式。
6. 根据当前对话中已经解决的问题，整理一条紧凑的新记录。
7. 写入前检查目标笔记文件是否已有相似内容。
8. 如果已有相似记录，不要重复追加，只告诉用户已有类似内容。
9. 如果没有重复内容，按照主题规则追加新记录。
10. 运行主题 `README.md` 要求的验证命令；如果没有定义验证命令，就不额外运行。
11. 如果验证失败，说明失败原因，不继续 commit 或 push。
12. 只有在用户明确要求 Git 持久化时，才执行 commit 或 push。

## 重复判断

判断是否相似时，重点看是否记录了同一主题、同一问题、同一关键结论、命令或配置。

## Git 规则

Git 操作是可选的，必须遵循用户请求和仓库自己的约定。

如果用户要求提交：

1. 先检查 `git status --short`。
2. 如果仓库配置了 remote，提交前先运行 `git pull --ff-only`。
3. 只提交本次学习记录相关文件。
4. commit message 要清楚说明新增了什么学习记录。

如果 pull、commit、push 或认证失败，停止并说明原因。不要套用任何机器专属的代理或网络绕过方案，除非用户明确要求。
